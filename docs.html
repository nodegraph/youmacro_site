Nodes


Nodes Perform Tasks

A node is represented visually by a rectangular shape with input and output shapes attached to it. A node shows its inputs as boxes and its outputs as upside down triangles.

When a node is executed it will look at the inputs on the node, perform some type of operation on them, and then place the results into its outputs. For example, there could be a node which adds two numbers together. In this case, the inputs would be the two numbers and the output would be the sum. (1+1=2)

The name of the node appears directly on the node and is user configurable. In this case, we see that the name is "data".

The data types that flow into an input or flow out of an output are modeled after javascript data types. The complete set of data types are numbers, strings, booleans, objects, and arrays.

Links

Links Direct Data Flow

A link is visually represented by an arrow where the base starts at an output of a node and the head ends at the input of another node.

A link directs data from the output one node to an input of another node. Each input expects a certain type of data. When the data types do not match, the upstream data is automatically converted to match. Then the upstream data is automatically merged into the expected input data type.

The merging process examines the internal properties of the data. Matching property names will get their value overridden with the upstream value. Non-matching property names will get their values injected into the input value.

Clean

Clean A Node To Run It

A node is said to be clean when all of its inputs and output values are up to date.

Cleaning a node causes the app to start performing work in order to make the node clean.

In order to compute an up to date output value, the node's inputs must be clean as well. Hence this is a recursive process which cleans all of the linked upstream nodes as well.

A clean node will be marked on the right with the letter "C". A dirty node will simply be missing the letter "C".


Edit


Edit A Node To Change Its Behavior

Editing a node involves changing the inputs to the node. This can be performed by rewiring the inputs to the node or editing the input value directly.

When an input is connected it cannot be edited directly as it's set to use the value from upstream.

However, when the input is not connected, it can be edited directly in the GUI through the data editor.

The current node being edited will be marked on the right with the letter "E".

An input is not always exposed with a box shape on the node. It can be explicitly hidden or exposed through the GUI. When an input is hidden it cannot be linked, but it can still be edited directly.


View

View A Node To Examine Its Outputs

Viewing a node, allows you to inspect the output values produced by the nodes in the data viewer.

Outputs of nodes can have one of these following data types: numbers, strings, boolean, objects, and arrays. The data viewer allows you to dive into objects and arrays by double clicking on them.

You can the outputs of both a dirty node and a clean node. The dirty node will show its current dirty values, while a clean node will show its current up to date values.

The current node being viewed will be marked on the right with the letter "V".


Groups

Groups Hide Complexity

A group node is a node which contains another node graph inside itself. When you dive into a group node, you will see the contents of the group node. When you surface from a group you will see the node graph in which the group node exists.

Group nodes allow us to treat a complex node graphs as one node. It also allows us to define a strict interface to the logic contained within. The interface is represented by the inputs and outputs on the group node.

In order to create inputs or outputs on a group node, you will first need to dive into the group. Then create Input Nodes or Output Nodes respectively. Each Input Node or Output Node inside the group will appear as inputs or outputs respectively on the group node under the same names.

There are a few different types of group nodes. Each provides a specialized context for their nodes to operate in.


Browser Groups Manage The Browser

The Browser Group Node automatically manages browser. Nodes which uses the browser must be created inside a Browser Group.

When diving into a Browser Group Node, a browser window will automatically open up. When surfacing from inside a Browser Group Node, the browser window will automatically get destroyed.

To create nodes which manipulate the browser you will first need to right-click inside the browser. This will bring up a menu containing possible nodes to create. The position of the right-click in the browser will be used to automatically set a number of input values on the nodes it creates (such as the type of the element or the text value of the element).

Custom Nodes

Create Custom Nodes With Javascript

Javascript is the language used to specify the script for our script nodes. Script nodes are very powerful, allowing users to essentially create brand new node types.

A script node only contains one input and one output. They are accessible in the scripting environment as "input" and as "output".

In the following example, the output's value property is being set to the sum of the input's x and y properties.


        output.value = input.x + input.y;

Script Nodes Inside While Loops

When the script node is placed inside a While node or another looping node, the script will often need to share state between iterations. The scripting environment provides a context object which will hold any properties between iterations. In the following example, the script node counts how many times it loops and sets it as the output's value property. 


        if (context.count == undefined) {
            context.count = 1
        } else {
            context.count += 1
        }
        output.value = context.count

Tips

Creating Browser Nodes

The following is the easiest way to create browser nodes. Watching some of the video demos will likely make things clearer.

  • First, create a Browser Group Node. Then dive into it.

  • Once you dive in, you'll see a browser window open up automatically. Wait while it loads the first web page.

  • Now right-click in the browser. A menu of possible browser nodes to create should open up in the app. Note that other nodes are usually created by right-clicking in the node graph instead.

  • The position of the right-click is significant as it allows the robot to automatically determine and set the input values on the nodes that it creates for you. This makes the creation of nodes much quicker. Otherwise, you'd need to manually edit each node to set appropriate values for the behavior you want, by yourself. Things like the element's text value, type or position are determined automatically.

  • Note that newly created nodes will automatically be connected to the last selected node. This reduces the amount of time you'll spend linking up nodes. So make sure you select the node you want to link to before creating a node.


Performing Actions on Elements

There are 3 basic steps to performing actions in the browser.

  • First, create a "Get All Elements" node, or identify an upstream node which outputs an array of web page elements. This array can contain just one element or it can contain many elements. An element for the robot is either an image, a piece of text, an input field, or a select dropdown. Note that you can visualize the elements at any time by feeding it into a Highlight Elements node.

  • Second, you'll likely want to filter down this array of elements to just the ones that you're actually interested in. To do this, link the previous node to any of the nodes which perform filtering operations on the array of elements. Isolate, prune, sort, join and remove are some common terms used in the names of these nodes.

  • Third, you need to perform an action on the elements you've filtered down. To do this, link the previous node to one of the action nodes, like Mouse Action. The MouseAction node will click the first element in the array of elements. Other nodes like the download videos node will download all the input elements.

By following the above tips, you will soon be creating powerful and time-saving macros.

YouMacro can automate it.
